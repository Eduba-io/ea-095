01  ·  The work this note addresses

Not the tools. The methodology layer above them.

The stack is built. Harvey, Microsoft Copilot, Kira, Aiscension with Reveal, Newcode on top. What lives above them, and what decides which tool handles which workflow, is a methodology question that currently sits inside the innovation function as institutional memory.

A practical view. About sixty percent of the daily work across a 4,300-lawyer platform is traditional code and database work: DMS, conflicts, time capture, billing, knowledge retrieval. Around thirty percent is rule-based logic: matter intake, jurisdictional rules for regulatory advice, approval workflows. About ten percent is a real AI problem: first-pass contract analysis with human review, cross-jurisdiction AI governance synthesis, client-facing AI risk assessments.

The tools in the firm's stack each cross multiple layers of that map. The methodology for deciding which tool lands on which layer (and what to bill the client for the output rather than the associate hour) is the work. Disciplined. Published. Built to be absorbed by the innovation team rather than delivered from outside.
